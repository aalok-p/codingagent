## Codebase Patterns
- Frontend API modules follow pattern: create file in `src/api/`, export functions using shared axios instance with interceptors for auth token
- Toast system: ToastContext + useToast hook in `src/components/Toast.jsx`, ToastProvider in `src/components/ToastProvider.jsx`, wraps app in App.jsx
- Product API uses `src/api/products.js` with axios interceptor to auto-attach Bearer token from localStorage
- Modal pattern: overlay + modal div with stopPropagation, form inside modal, close on overlay click
- Stock badge CSS classes: stock-high (green, >10), stock-medium (yellow, 6-10), stock-low (red, <=5)
- Frontend pages in `src/pages/` directory, subdirectories per role (e.g. `src/pages/seller/`)
- ESLint rules enforced: react-refresh/only-export-components (separate context from component), react-hooks/set-state-in-effect (no setState synchronously in useEffect), react-hooks/refs (no ref.current assignment during render)
- Use `app/utils/security.py` for JWT token creation and password hashing/verification
- Use `app/schemas/` directory for Pydantic request/response models
- Auth routes use `/auth` prefix via APIRouter, registered in main.py
- Password validation uses Pydantic `@field_validator` on the schema class
- JWT payload: `{"sub": str(user_id), "role": user_role, "exp": expires}`
- TokenResponse schema includes access_token, token_type, user_id, role
- Use `app/dependencies.py` for shared FastAPI dependencies (e.g., `get_current_user`)
- Protect routes with `get_current_user` dependency from `app/dependencies.py`
- Google OAuth new users get `role="pending"` until they choose buyer/seller via PATCH /auth/role
- CORS middleware configured in `app/main.py` with broad defaults
- Frontend: `src/api/auth.js` for all auth API calls; `setAuthToken()` to configure axios default header
- Frontend: role is passed as query param `/login?role=buyer|seller` to pre-select in forms
- Frontend: after login/signup, store token/userId/role in localStorage; redirect based on role
- Frontend: Google OAuth via `@react-oauth/google`; client ID from `VITE_GOOGLE_CLIENT_ID` env var
- Frontend: role-pending users (Google OAuth first login) redirected to `/role-picker` page
- Product CRUD routes use `/products` prefix via APIRouter in `app/routers/products.py`
- New routers follow pattern: create APIRouter in `app/routers/`, register via `app.include_router()` in `app/main.py`
- Role-based access control: check `current_user.role` in endpoint logic for seller-only or buyer-only routes
- SUI number generation: `random.randint(10**7, 10**12 - 1)` with uniqueness check against DB
